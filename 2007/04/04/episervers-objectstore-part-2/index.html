EPiServer's ObjectStore (Part 2)

April 4, 2007 by Adam Najmanowicz 6 min read
CMS EPiServer Internet Information Services Software Development

I think we’re mostly finished investigating ObjectStore for now. In this article I’ll try to finish up on the apsects of using the Object store in a real life solution that is a basic Page comments. In my previous article concerning ObjectStore I have described a way of storing and retrieving an object from the Store, which is fine and dandy if we know exactly the object’s OD, for example if we reference it from a page. But what good is a store like that if we cannot search it for content? The problem we were trying to solve using ObjectStore was storing comments for ANY Episerver page without having to do anything to the page type. We might need that for the upcoming project so the discovery may prove useful since this is a really neat way of storing objects.

So the class that we are going to store needs to persists the following values:

 1namespace Cognifide.EPiServerTest.ObjectStore
 2{
 3    [Serializable, XmlInclude(typeof (PageCommentDO))]
 4    public class PageCommentDO : IItem, IPageComment
 5    {
 6        private object id;
 7        private string name;
 8        [Indexed(true)]
 9        private int pageId;

18        [Indexed(true)]
19        private bool reported;

21    }

 You might have noticed that contrary to what I did before this class has some of its fields tagged with [Indexed(true)] attribute. This is required by the store object if we ever want to query the store for the class using those fields as a filtering criteria. The page also implements the required IItem interface (although the properties required by the interface are not shown in the excerpt) as well as our internal IPageComment which is added there for the purpose of easy switching of implementations. We intend to implement it on both ObjectStore and using NHibernate to make sure we get the best performance possible out of our solution. This interface is to allow us to easily switch between the 2 implementations.

Back to the ObjectStore

 First we need to introduce a few new classes EPiServer uses for querying the Store. The namespace that scopes the querying consists of the following entities:

The Query class is at the center of any search in the Store, let’s start with the samples right away. The saving is almost identical to the sample that I’ve shown in my previous article:

Really there is not much meat there. The interesting part comes in the object retrieval. Even though the Search namespace does not look like it offers much, you can still build a fairly fophisticated filter with it. following code retrieves comments for a specific PageLinkID. The comments can be filtered by date and since the call is used to retrieve the page for viewing, we probably only want the ones that are public (meaning they have not been removed by the comment moderator).

 1public static List<PageCommentDO> GetCommentsForPage(int pageId, 
 2    DateTime beforeDate, DateTime afterDate, bool includeOnlyPublished)
 3{
 4    ISession session = null;
 5 
 6    try
 7    {
 8        // make sure there is a schema to read from, otherwise there is no point
 9        if (Context.Repository.SchemaForType( typeof (PageCommentDO)) == null)
10        {
11            return null;
12        }
13 
14        //acquire a session
15        session = Context.Repository.CreateSession();
16 
17        //create the query 
18        Query query = new Query(typeof (PageCommentDO));
19 
20        // we only want comments for a singler page 
21        query.Add(Expression.Equal("pageId", pageId));
22 
23        // should we filter to only wshow comments from some set time period?
24        // the following demonstrate how to request a value from within a range
25        if ((beforeDate != DateTime.MinValue) ||
26            ((afterDate != DateTime.MaxValue) && (afterDate != DateTime.MinValue)))
27        {
28            query.Add(
29                Expression.Between("submitDate",
30                                   (beforeDate != DateTime.MinValue) ? beforeDate : new DateTime(0x76c, 1, 1),
31                                   ((afterDate != DateTime.MinValue) && (afterDate != DateTime.MaxValue))
32                                       ? afterDate
33                                       : new DateTime(0x834, 1, 1)));
34        }
35 
36        // probably for a displayed page we will only want to show published comments
37        if (includeOnlyPublished)
38        {
39            query.Add(Expression.Equal("published", true));
40        }
41 
42        // order the comments by date/time so that we get the most recent first
43        query.AddOrder(new Order("submitDate", true));
44 
45        IList list = session.ExecuteQueryObject(query);
46        List<PageCommentDO> result = new List<PageCommentDO>(list.Count);
47        foreach (object item in list)
48        {
49            result.Add((PageCommentDO) item);
50        }
51        return result;
52    }
53    catch (ElektroPostException ex)
54    {
55        // ... do some reporting
56        throw;
57    }
58    finally
59    {
60        if (session != null)
61        {
62            session.Close();
63        }
64    }
65}

You nahe to agree, this really IS neat. All you need to do now to place the comments on a page is a few lines of code to bind the data from the Store with a repeated of a kind, and a set of edit boxes with a submit button :)

This is basically how much code the submission and retrieval the comments comment take:

 1// Load the comments to a repeater
 2protected void Page_Load(object sender, EventArgs e)
 3{
 4        List<PageCommentDO> comments = 
 5            PageCommentDO.GetCommentsForPage(CurrentPage.PageLink.ID, DateTime.MinValue, DateTime.MaxValue);
 6        CurrentComments.DataSource = comments;
 7        CurrentComments.DataBind();
 8}
 9 
10// Comment submission
11protected void SubmitComment(object sender, System.EventArgs e)
12{
13    PageCommentDO newComment = new
14        PageCommentDO(Guid.Empty,
15        string.Format("CommentForPage{0}", CurrentPage.PageLink.ID),
16        CurrentPage.PageLink.ID,
17        SubjectTextBox.Text, ContentTextBox.Text, DateTime.Now, false, true, false);
18    newComment.Save();
19}
20 
21// the data binding for the labels
22protected void CurrentComments_ItemDataBound(object sender, RepeaterItemEventArgs e)
23{
24    PageCommentDO comment = (e.Item.DataItem as PageCommentDO);
25 
26    Label commentSubjectLabel = (Label)e.Item.FindControl("CommentSubjectLabel");
27    if (comment != null)
28    {
29        commentSubjectLabel.Text = comment.Title;
30    }
31 
32    Label commentContentLabel = (Label)e.Item.FindControl("CommentContentLabel");
33    if (comment != null)
34    {
35        commentContentLabel.Text = comment.Content;
36    }
37}
